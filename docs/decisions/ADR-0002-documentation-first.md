# ADR-0002: Documentacion primero

## Contexto

El proyecto manejara muchas integraciones, credenciales, automatizaciones y decisiones tecnicas.

## Decision

Toda capacidad importante debe tener documentacion antes o junto con su implementacion.

## Consecuencias

- Menos ambiguedad.
- Mejor transferencia de conocimiento.
- Decisiones revisables.
- Mayor disciplina tecnica.

## Alternativas consideradas

Documentar despues de implementar. Se descarta porque suele generar deuda documental.
