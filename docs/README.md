# Documentation

La documentacion es parte central de RRSS AUTO.

Este directorio explica arquitectura, decisiones, modulos, operacion, seguridad, pruebas, IA, automatizacion e integraciones.

Toda decision importante debe registrarse antes o junto con su implementacion.

## Capas documentales

- `architecture/`: arquitectura general.
- `domain/`: diseno DDD del dominio central.
- `rfc/`: propuestas fundacionales de subsistemas.
- `blueprints/`: flujos funcionales de negocio que guian implementacion futura.
- `specifications/`: contratos de ingenieria para implementacion futura.
- `decisions/`: decisiones arquitectonicas aceptadas.
